\lhead[\fancyplain{}{\bfseries\thepage}]{\fancyplain{}{\bfseries\rightmark}}

This chapter introduces the $\lambda$-BLL calculus~\cite{lago2024computationalindistinguishabilitylogicalrelations}, a typed probabilistic $\lambda$-calculus to model polynomial-time cryptographic computation. Its design principles and language features capture the essential for expressing cryptographic experiments, adversaries, and reductions.

$\lamBLL$ provides a programming-language framework in which computational complexity is enforced statically by the type system. In particular, it supports probabilistic polynomial-time computation, bounded interaction with oracles, shared mutable state and higher-order reasoning about cryptographic constructions. These features make it well-suited for formalizing experiment-based cryptographic proofs.

The type system of $\lambda$-BLL is linear and graded, with a close correspondence to Bounded Linear Logic and graded calculi. Resource usage is tracked by polynomial grades, generated by:
\begin{align*}
p ::= 1 \mid i \mid p + p \mid p * p
\end{align*}
where the variable $i$ represents the security parameter.

The calculus follows a call-by-push-value (CBPV) discipline~\cite{levy2012call}, distinguishing between positive types and general types. This separation permits to restrict function application to positive arguments only.

The base of the type system consists of ground types:
\begin{align*}
\UU \mid \BB \mid \SS[p] 
\end{align*}
representing unit, booleans, and fixed-length bitstrings of polynomially bounded size. These types correspond to the fundamental data manipulated in cryptographic protocols, such as nonces, messages, and keys.

Positive types extend ground types with:
\begin{align*}
P \otimes P \mid \bang_p A 
\end{align*}
namely pairs formed with the tensor products $P \otimes P$, and the graded comonadic modality $\bang_p A$. In a linear type system, a value must normally be used exactly once. The modality $\bang_p A$ relaxes linearity by allowing a value of type $A$ to be used up to $p$ times. This is central to the cryptographic interpretation of the calculus because it enables a static enforcement of polynomial-time bounds. For example, an oracle of type $\bang_{q(i)} \Oracle$ may be queried at most $q(i)$ times by an adversary for a polynomial $q$. Any attempt to exceed this bound results in an ill-typed term, ensuring that polynomial-time constraints are enforced by typing rather than by operational reasoning. As a result, the proof fails at the type level rather than at runtime.

$\lamBLL$ extends the pure lambda-calculus with other features, necessary for modelling cryptographic computation, for example it defines a collection of function symbols $\mathcal{F}$. Each function symbol $f \in \mathcal{F}$ has a type denoted by $\mathsf{typeof}(f)$ of the form $G_1 \times \dots \times G_m \to G$ where $G_1, \dots, G_m$ and $G$ are ground types. Then, for each polynomial $p$, in $\lamBLL$ there is a corresponding constructor $f_p$ representing a resource-bounded invocation of $f$, with the grade $p$ accounting for its computational cost within the type system.

Values represent data that can be passed, stored or paired but not directly executed, while computations represent effectful execution. Ground values include $\star$, booleans, and bitstrings. Positive values include variables, pairs, and suspended computations of the form $!M$. Computations follow the CBPV structure so values are injected using $\return$, suspended computations are forced using $\der$, and applications is defined only on positive arguments.

To model cryptographic experiments involving shared state, $\lamBLL$ includes global references. References store ground values (and ground only) and are manipulated via $\setr{r}{Z}$ and $\get r$. This ensures that stateful effects do not introduce higher-order behavior beyond what is explicitly admitted by the type system. The language provides also control constructs such as conditionals, let-bindings, and pattern matching on pairs. Bounded iteration is expressed using the construct $\loopp{V}{p}{M}$, which executes the computation $M$ at most $p$ times.

Since variables in $\lamBLL$ are classified according to their types, a variable is said to be ground if its type contains no function arrows, preventing ground terms from encoding higher-order behavior thus being dangerous as global references. To track higher-order dependencies, the calculus distinguishes the set of free higher-order variables of a term $M$, denoted $\mathit{HOFV}(M)$. This set contains exactly those free variables whose types include at least one occurrence of the function type constructor $(\multimap)$.

$\lamBLL$ considers two different typing contexts
\begin{align*}
\Gamma &::= \emptyset \mid \Gamma, x: P \tag{Variable context} \\
\Theta &::= \emptyset \mid \Theta, r: P \tag{Reference context}
\end{align*}
The context $\Gamma$ contains term variables with positive types only, which are not computations, while the reference context $\Theta$ contains references which are can only be ground types. Fixed a reference context $\Theta$ then there are two typing judgments with respect to $\Theta$.
\begin{align*}
\Gamma &\vdash_v^\Theta V: A \tag{Value typing} \\
\Gamma &\vdash_c^\Theta M: A \tag{Computation typing}
\end{align*}
Computations may access references, but references never contain computations.

The polynomial addition induces a binary partial operation $\boxplus$ on positive types defined by induction
\begin{align*}
	G \boxplus G &:= G\\
	(P \otimes Q) \boxplus(R \otimes S) &:= (P \boxplus R) \otimes (Q \boxplus S) \\
	(\bang_p^{\Theta} A) \boxplus (\bang_q^{\Theta}  A) &:= \bang_{p+q}^{\Theta}  A
\end{align*}
The partial operation $\boxplus$ can be extended as a total operation on contexts
\begin{align*}
	\emptyset \boxplus \emptyset&:= \emptyset \\
	(\Gamma, x:P) \boxplus \Delta & := \begin{cases}
		\Gamma, x: P \boxplus \Delta & \text{ if } x \text{ does not occur in } \Delta \\
		x : P \boxplus Q, \Gamma \boxplus \Sigma & \text{ if } \Delta = \Sigma, x: Q
	\end{cases}
\end{align*}

And to account for polynomial multiplication, for every polynomial $p \in \mathbb{N}_{\geq 1}[i]$ there is a total unary operation on positive types by induction:
\begin{align*}
	p * G &:= G\\
	p * (P \otimes Q)  &:= (p * P) \otimes (p * Q) \\
	p * (\bang_q^{\Theta}  A) &:= \bang_{p\times q}^{ \Theta}  A
\end{align*}

The operation $(p\,*)$ on positive types can also be extended to a total operation on term variables contexts:
\[\begin{aligned}
	p \ast \varnothing &:= \varnothing\\
	p \ast  (x : P,\Gamma )&:= (x : p \ast P),  p\ast \Gamma 
\end{aligned}\]

The notion of substitution in $\lamBLL$ is defined with respect to the distinguished security parameter $i$. Given a polynomial $p \in \mathbb{N}_{\geq 1}[i]$ and a type $A$, the type $Ap$ is the type obtained by substituting every occurrence of the parameter $i$ in $A$ with the polynomial $p$, that is, $A[i:=p]$. Similarly, for a term $M$, the type $Mp$ to denote the term obtained by substituting $p$ for $i$ in all polynomial annotations occurring in $M$, namely $M[i:=p]$. The substitution acts only on polynomial indices appearing in types and terms, and does not affect term variables or computational structure.

A computation in $\lamBLL$ can be probabilistic and can read or modify the global reference context. In order to model probabilistic effects together with mutable references, $\lamBLL$ combines the distribution monad with the state monad. The key idea is that the result of a computation is not just a value: it depends on the initial store, it produces a distribution of possible outcomes, and each outcome is paired with an updated store. Accordingly, computations are interpreted as functions that take an initial store and return a probability distribution over pairs consisting of a value and a store.

To account for probabilistic effects, $\lamBLL$ models probabilistic computation using finite probability distributions. Given a set $X$, a probability distribution over $X$ is a function $\mu : X \to [0,1]$ with finite support and total mass equal to $1$. Any such distribution can be written as a finite weighted sum of Dirac distributions $\delta_x$, which represent deterministic outcomes. This construction induces a monad $(\D, \eta_{\D}, \bind_{\D})$ on the category $\mathbf{Set}$ of sets and functions. The unit maps a value $x$ to the Dirac distribution $\delta_x$, embedding pure values into probabilistic computations having components $\eta_X : x \mapsto \delta_x$, while the bind operation models sequential probabilistic computation by combining distributions:
\[
\bind : \D(X) \times \mathbf{Set}(X,\D(Y)) \to \D(Y).
\]
Here, $\D(X)$ denotes the set of all probability distributions over $X$.

To model mutable references, $\lamBLL$ uses a state monad whose state space is given by the set
of stores associated with a fixed reference context $\Theta$. A store assigns to each reference
a ground value of the appropriate type, thereby ensuring that memory remains well-typed
throughout execution. Reading a reference retrieves its current value from the store, while
writing to a reference produces a new store in which the corresponding location is updated.

The full semantics of computations in $\lamBLL$ is obtained by combining probabilistic choice with state. For a fixed reference context $\Theta$, computations returning values in a set $X$ are interpreted as functions from stores to probability distributions over pairs of values and stores. For every closed reference context $\Theta$, there is a corresponding monad $(\T_\Theta, \eta_\Theta, \bind_\Theta)$ on $\Set$, defined as the tensor product of the distribution monad with the state monad, namely
\[
\T_{\Theta} := \bigl(\D(X \times \Store_{\Theta})\bigr)^{\Store_{\Theta}}
\]
The unit of $\T_\Theta$ has components $\eta_\Theta : X \to \D(X \times \Store_\Theta)^{\Store_\Theta}$, mapping a value $x \in X$ and a store $e \in \Store_\Theta$ to the Dirac distribution
$\delta_{(x,e)}$, corresponding to a deterministic computation that returns $x$ without
modifying the store. The bind operator
\[
\bind_\Theta : \T_\Theta X \times \mathbf{Set}(X, \T_\Theta Y) \to \T_\Theta Y
\]
maps $\varphi \in \T_\Theta X$ and $f : X \to \T_\Theta Y$ to the function
$\dlambda e.\, \bigl(\varphi(e) \bind_\D \eval \circ (f \times \id_{\Store_\Theta})\bigr)$,
where $\dlambda$ denotes currying and $\eval$ is the evaluation map induced by the Cartesian closed structure of $\Set$.

In $\lamBLL$, application is written as $MZ$, and the constructor $\return$ is used to lift values into computations. The syntactic sugar
\begin{align}
    VZ \triangleq (\return V)\,Z\label{eq:sugar-return},
\end{align}
allows to simplify expressions by implicitly promoting values to computations, thereby reducing the need for nested $\return$ construct.

\section{Lambda-BLL Extension for Proving MAC Security}\label{sec:extensions-lambda-bll}
We introduce a formal system of Hoare Logic \cite{axiomatic-basis-for-computer-programming}, which will be required to establish the existential unforgeability under chosen-message attacks (EUF-CMA) of PRF-induced MACs within the $\lamBLL$ framework. Following Hoare's original formalism, we describe program behavior by means of \emph{partial correctness specifications}, written as triples of the form
\[
\{P\} S \{Q\}.
\]
Here, $S$ denotes a program, and $P$ and $Q$ are predicates representing, respectively, the precondition and postcondition. The intended semantics of this triple are that, if the execution of $S$ is started in any state that satisfies $P$, then every terminating execution of $S$ will result in a state that satisfies $Q$.

A \emph{state} is defined as an assignment of values to all program variables. Predicates over states are written using standard mathematical notation and logical connectives, thereby characterizing sets of such states. The specification is termed \emph{partial} because the validity of $\{P\} S \{Q\}$ does not require that the execution of $S$ actually terminates when initiated in a state satisfying $P$, it only constrains the states reached upon termination, should termination occur. 

A stronger notion, referred to as \emph{total correctness specifications}, is written as $[P] S [Q]$. This triple asserts that whenever $S$ is executed from a state satisfying $P$, (i) the execution of $S$ terminates, and (ii) the resulting state satisfies $Q$. In the setting considered here, we assume that all programs $S$ terminate.

Hoare Logic is essentially a deductive proof system for reasoning about Hoare triples $\{P\} S \{Q\}$. Let $\mathcal{R}(\Phi)$ denote the set of program variables (or references) that occur in a predicate $\Phi$. Using this notation, we formulate the following inference rules presented in Figure~\ref{fig:hoare-rules} that instantiate a simple Hoare Logic for the $\lamBLL$ calculus. In particular, several rules play a crucial role in formalizing the proof of Theorem~\ref{theorem:mac-security}.

The \textit{reflexivity} rule (\textsc{H-REFL}) rule captures the reflexivity of the logic, stating that any expression $M$ preserves a predicate $\Phi$ when it is related to itself. The \textit{consequence rule} (\textsc{H-CONS}) enables the strengthening of preconditions and the weakening of postconditions. The \textit{let rule} (\textsc{H-LET}) rule formalizes sequential composition, chaining the transition from an initial precondition $\Phi$ to a final postcondition $\Psi$ via an intermediate predicate $\Theta$. The \textit{if rule} (\textsc{H-IF}) rule handles conditional branching: it requires that both branches of a conditional expression establish the postcondition $\Psi$ under their respective boolean constraints. The rules \textsc{H-THEN} and \textsc{H-ELSE} constitute the axiomatic basis for reasoning about conditionals when the guard $b$ is known to evaluate to true or false, respectively. The \textit{preservation rule} (\textsc{H-PRES}) states that a predicate $\Phi$ remains invariant provided that the set of resources $\mathcal{R}(M)$ modified by the expression $M$ does not intersect the set of resources $\mathcal{R}(\Phi)$ referenced by $\Phi$. Finally, the \textsc{H-Frame} rule allows the extension of both the precondition and the postcondition with an additional predicate $\Theta$, under the requirement that $\Theta$ is disjoint and does not mention any resources modified by $M$ or $N$.
