\chapter{Proving PRF-Induced MAC Security Equationally}
\lhead[\fancyplain{}{\bfseries\thepage}]{\fancyplain{}{\bfseries\rightmark}}
This chapter demonstrates that existential unforgeability under chosen-message attacks for pseudorandom function based message authentication codes may be defined and validated inside the $\lamBLL$ framework. In particular, we formalize the standard reduction from MAC forgery to pseudorandom function
indistinguishability by encoding the $\macexp$ experiment, the adversary, and the
MAC oracle as $\lamBLL$ terms, and by reconstructing the reduction outlined in
Section~\ref{subsec:mac-security} using the equational reasoning principles of the
calculus. This formalization demonstrates that $\lamBLL$ is sufficiently expressive
to internalize classical cryptographic security reductions while enforcing polynomial-time constraints through its graded type system.

We begin by recalling the MAC-forgery experiment $\macexp$ and its pseudocode,
reported in Figure~\ref{fig:macforge-recall}, and by specifying the types of the
components of the experiment as they appear as subterms in the $\lamBLL$
formalization. The concrete implementation of the terms modeling the key-generation
algorithm $Gen$ and the MAC oracle $Mac_k(\cdot)$ depends on the specific MAC
scheme under consideration. For this reason, these terms are left abstract and will be instantiated only after fixing a scheme $\Pi$. In the following
paragraphs, we make this instantiation explicit for the schemes $\Pi_F$ and
$\widetilde{\Pi}$ that are used in the proof of Theorem~\ref{theorem:mac-security}.

\begin{figure}[htpb]
    \centering
    \begin{align*}
    &\macforge(n): \\
    &\quad k \gets Gen(1^n) \\
    &\quad (m,t) \gets A^{Mac_k(\cdot)}(1^n) \\
    &\quad \mathbb{Q} \gets \{m \ | \ A \ \text{queries} \ Mac_k(m) \} \\
    &\quad \mathtt{return}\,\,(m \notin \mathbb{Q} \land \vrfy_k(m,t) = 1)
    \end{align*}
    \caption{Pseudocode for $\mathtt{MacForge}$ experiment}
    \label{fig:macforge-recall}
\end{figure}

We now describe how the components of the experiment are typed in $\lamBLL$.
The key-generation algorithm $Gen$ is modeled as a probabilistic computation that,
given the security parameter, produces a secret key whose length is polynomial in $i$. The adversary is modeled as a higher-order computation that receives oracle access to the MAC function and outputs a candidate forgery. The MAC oracle itself is represented as a term whose use is controlled by a graded modality, enforcing a polynomial bound on the number of queries.

\begin{align*}
&\vdash_v^\Theta Gen: \UU \multimap \SS[i] \\
&\vdash_v^\Theta Mac: (\SS[i] \otimes \SS[i]) \multimap \SS[i] \\
&\vdash_v^\Theta \vrfy: (\SS[i] \otimes \SS[i]) \multimap \BB \\
&\vdash_c^\Theta Oracle: \SS[i] \multimap \bang_{q}\bigl(\SS[i] \multimap \SS[i]\bigr)\\
&\vdash_c^\Theta Adv: \bang_{q}\bigl(\SS[i] \multimap \SS[i]\bigr) \multimap (\SS[i] \otimes \SS[i])
% \vdash_c^\Theta &: 
% \bang_{q}\bigl(\SS[i] \multimap \SS[i]\bigr) \multimap \bigl((\SS[i] \otimes \SS[i]) \otimes \SS[r]\bigr) && \tag{Type for $Adv$}
\end{align*}

The typing judgments for the main components of the experiment are given above.
Here, $i$, $r$, and $q$ denote polynomials in the security parameter. The oracle type $\SS[i] \multimap \bang_{q}(\SS[i] \multimap \SS[i])$ models a MAC oracle which, once instantiated with a key, provides a function mapping messages of length $i$ to tags of the same length. The graded modality $\bang_{q}$ enforces a polynomial bound on the number of oracle queries that the adversary may perform. The oracle is responsible not only for answering MAC queries, but also for recording all messages queried by the adversary. This information is stored in the reference context $\Theta$, which is updated at each oracle invocation and thus maintains encoding of the adversary's query history. The adversary $Adv$ takes as input the closure obtained by currying and encapsulating the key within the $Oracle$ term. Based on its interaction with the oracle, it produces a candidate forgery $(m,t)$ having type $(\SS[i] \otimes \SS[i])$. The verification function $\vrfy : (\SS[i] \otimes \SS[i]) \multimap \BB$ checks the validity of a proposed tag by recomputing the MAC on the message component and comparing it with the supplied tag. In addition to answering MAC queries, the oracle maintains a record of all messages queried by the adversary. This information is stored in the reference context $\Theta$ as a reference $Q$ representing the set
\[
\mathbb{Q} \gets \{\, m \mid A \text{ queries } Mac_k(m) \,\}.
\]
The reference $Q$ has type $\SS[r]$, where $r = q \cdot i + 1$, and encodes the sequence of messages submitted to the oracle. At each oracle invocation, the reference is updated to include the fresh message. To inspect the adversary's query history, we introduce an auxiliary predicate $\mathtt{contains}$, whose purpose is to determine whether a given message occurs in the encoding of the set $\mathbb{Q}$ stored in the reference context. More precisely, $\mathtt{contains}$ checks whether a bitstring of type $\SS[i]$ occurs in the encoding of a longer bitstring of type $\SS[r]$ representing the query history. The type of the predicate is given by
\[
\vdash^\Theta_v\ \mathtt{contains} : (\SS[r] \otimes \SS[i]) \multimap \BB,
\]
indicating that it takes as input the stored query history together with a candidate message and returns a boolean value. The reference $Q$ stored in the context $\Theta$ is updated by the oracle at each query by executing $\mathtt{set}\,Q$ assigning the value $\mathtt{concat}(\get Q, m)$, where $m$ is the freshly queried message. The predicate $\mathtt{contains}$ reads the reference context and checks whether there exists an encoded query exactly equal to its message argument.

It is worth noting that the type derivations of the terms introduced so far are parameterized by a reference context $\Theta$, which varies depending on the cryptographic experiment considered. In the model for the scheme $\Pi_F$, the context $\Theta$ contains the reference $Q$ used to record oracle queries.
In contrast, in the model for $\widetilde{\Pi}$, the context contains a reference used to model the random function $f$.

We define the $\lamBLL$ term corresponding to the forgery experiment, shown in
Figure~\ref{fig:macforge-term}. The experiment begins by generating a fresh key $k$ and constructing the corresponding oracle $o$. The adversary $Adv$ is then given access to $o$ and returns an attempted forgery $(m,t)$. The experiment first checks whether the verification procedure $\vrfy$ accepts the pair $(m,t)$. If verification succeeds, it tests whether the message $m$ appears in the query set $\mathbb{Q}$ by reading the reference $Q$ and applying the predicate $\mathtt{contains}$. The experiment returns $\true$ if $m$ was not previously queried, and $\false$ otherwise. If verification fails, the experiment immediately returns $\false$.

\begin{figure}[ht]
    \centering
    \begin{align*}
    &MacForge \triangleq \\
    &\quad \letin{k = Gen \star}{} \\
    &\quad \letin{o = Oracle\, k}{} \\
    &\quad \letin{\langle m, t \rangle = Adv\, o}{} \\
    &\quad \letin{queries = \get Q}{} \\
    &\quad \letin{b = \vrfy\, \langle m, t \rangle}{} \\
    &\quad \mathtt{if}\, b\, \mathtt{then}\, \mathtt{not}(\mathtt{contains}(queries, m)) \\
    &\quad \mathtt{else}\, \false
    \end{align*}
    \caption{$\lamBLL$ term for the $\mathtt{MacForge}$ experiment.}
    \label{fig:macforge-term}
\end{figure}

We have to define a $\lamBLL$ term for the experiment using the encryption scheme $\Pi_F$ which in Proof~\ref{proof:handwavy} corresponds to the experiment $\mathtt{MacForge}_{A, \Pi_F}$ and one for the experiment using the encryption scheme $\widetilde{\Pi}$ which corresponds to $\mathtt{MacForge}_{A,\widetilde{\Pi}} $. In both the cryptographic experiment and the distinguisher, we need to construct two instances of the $\lamBLL$ term $\mathit{MacForge}$ (Figure~\ref{fig:macforge-term}) that models these components, namely one that interacts with the pseudorandom function $F$ and one that interacts with a truly random function $f$.
